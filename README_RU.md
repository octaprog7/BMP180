# BMP180

MicroPython модуль для датчика давления и температуры BMP180.

# Внимание
## Важное замечание
Внимание, адепты ATtiny, повелители ассемблера и Си! Предупреждение: этот проект может вызвать у вас неконтролируемые приступы гнева и ярости! 
Код написан на MicroPython и намеренно игнорирует следующие заповеди bare-metal разработки:

    «Каждый байт на счету» -> здесь ~8 КБ RAM? Не проблема.
    «Прямой доступ к регистрам» -> здесь set_*(...) с защитой от дурака.
    «Такты считать обязательно»-> здесь time.sleep_ms() и пусть железо подождёт.

Если вы только что почувствовали приступ гнева — поздравляем, вы попали по адресу.
Но если вам нужно быстро, надёжно и с единым API для BMP180/280/390 — добро пожаловать в мир, где код читается как книга.

# Подключения
Подключите плату BMP180 к Arduino, ESP или любой другой плате с прошивкой MicroPython.

Питание BMP180: 3,3 В! Не 5 В! Используйте четыре провода для подключения:

1. +3,3 В
2. GND
3. SDA
4. SCL

Загрузите прошивку MicroPython на плату NANO (ESP и т. д.), а затем файлы: main.py, bmp180.py и папку sensor_pack_2 полностью.

Затем откройте main.py в вашей IDE и запустите его.

# Измерение высоты над уровнем моря по давлению воздуха
Давление падает с высотой - значит, можно измерить высоту по давлению! Верно. Но есть проблемы!
## Физика процесса
Атмосферное давление уменьшается с высотой потому что над вами становится меньше столб воздуха. Зависимость нелинейная и зависит от:
* Температуры воздуха (тёплый воздух 'легче')
* Влажности (влажный воздух легче сухого)
* Широты и силы тяжести (g меняется от экватора к полюсам)
* Погодных фронтов (циклоны/антициклоны меняют давление на уровне моря)
## Барометрическая формула (упрощённая)
Для практических расчётов в диапазоне 0..5000 м используется международная формула:

```code
def pressure_to_altitude(pressure_pa: float, sea_level_pa: float = 101325.0) -> float:
    """
    Расчёт высоты над уровнем моря по давлению.
    
    :param pressure_pa: Измеренное давление в Паскалях
    :param sea_level_pa: Давление на уровне моря (QNH) в Паскалях
    :return: Высота в метрах (относительно заданного QNH)
    
    Точность: ±3..5 м при идеальных условиях.
    В помещении - результат будет неправилен!
    """
    return 44330.0 * (1.0 - (pressure_pa / sea_level_pa) ** 0.1903)
```
### Постоянные
| Постоянная | Ед. изм. | Физический смысл                                                                                     |
|------------|----------|------------------------------------------------------------------------------------------------------|
| 44330.0    | метры    | Масштабная высота атмосферы при стандартных условиях.                                                |
| 1903       | безразм. | (g·M) / (R·L) - комбинация гравитации, молярной массы, газовой постоянной и температурного градиента |

### QNH (давление на уровне моря)
Без актуального QNH расчёт высоты бессмысленен!

Пример:
* Измерено: 95000 Па
* При QNH = 101325 Па → высота ≈ 557 м
* При QNH = 100000 Па → высота ≈ 673 м  (! разница 116 м)

### Где взять QNH?
* Метеосайты
* Ручная калибровка по известной высоте
* GPS приемник

### Ручная калибровка (если известна высота места)

```code
def calibrate_qnh(measured_pressure_pa: float, known_altitude_m: float) -> float:
    """
    Вычисляет локальное QNH по известной высоте и измеренному давлению.
    
    :param measured_pressure_pa: Давление, измеренное датчиком (на улице!)
    :param known_altitude_m: Точная высота места над уровнем моря (по карте)
    :return: QNH в Паскалях для последующих расчётов
    """
    # Обратная барометрическая формула
    return measured_pressure_pa / (1.0 - known_altitude_m / 44330.0) ** (1.0 / 0.1903)


# Пример использования:
# Вы на даче, высота по карте: 185 м
# Датчик (на улице, в тени) показывает: 99120 Па
qnh = calibrate_qnh(99120, 185)  # ≈ 101280 Па
# Теперь можно считать высоту относительно этой точки:
current_alt = pressure_to_altitude(99050, qnh)  # ≈ 191 м (поднялись на 6 м)
```
Калибровку проводите только на открытом воздухе, в безветренную погоду, после 10-минутного прогрева датчика!!!

### Температурная компенсация (для точности 1 м)
Упрощённая формула (calibrate_qnh) предполагает стандартную температуру +15 гр. Ц. на уровня моря и градиент −6.5 гр. Ц/км. 
Для повышения точности можно учитывать реальную температуру:

```code
def pressure_to_altitude_temp(pressure_pa: float, temperature_c: float, 
                              sea_level_pa: float = 101325.0) -> float:
    """
    Расчёт высоты с учётом температуры воздуха.
    
    :param pressure_pa: Измеренное давление, Па
    :param temperature_c: Температура воздуха у датчика, гр. Ц.
    :param sea_level_pa: QNH, Па
    :return: Высота, м
    """
    # Базовая высота по упрощённой формуле
    h0 = 44330.0 * (1.0 - (pressure_pa / sea_level_pa) ** 0.1903)
    
    # Поправка на отклонение температуры от стандартной (15 гр. Ц.)
    # Формула: h = h0 * (T_actual / T_standard)
    t_actual_k = temperature_c + 273.15
    t_standard_k = 288.15  # 15 гр. Ц. в Кельвинах
    
    return h0 * (t_actual_k / t_standard_k)
```
При +30 гр. Ц вместо +15 гр. Ц. поправка составит ~ +5 % (на 100 м высоты - +5 м). 
Для любительских задач часто пренебрежимо.

# Высота по давлению: для самых занятых
## Хочу мерить высоту барометром. Как?
Инструкция без воды, формул и умных слов:
* Дома высоту не меряй — будешь мерить сквозняки.
* На улице — можно, но с условиями.
### Что нужно
* Датчик (BMP180/BME280)
* Микроконтроллер
* Улица
## Алгоритм в 4 шага
1. Выйди на улицу (серьёзно).
2. Узнай давление на уровне моря (QNH) — например, на rp5.ru.
3. Вставь в формулу: высота = 44330 × (1 − (твоё_давление / QNH)^0.1903)
4. Получи примерную(!) высоту. Не верь знакам после запятой.
## Формула 'на пальцах'
```code
# Скопировал, вставил
def height(pressure_pa, qnh_pa=101325):
    return 44330 * (1 - (pressure_pa / qnh_pa) ** 0.1903)

# Пример:
# Давление: 95000 Па, QNH: 101325 Па
# высота примерно 557 метров
# Если не знаешь QNH — подставь 101325. Ошибка будет до +/- 250 м!
```
## Чего не делать
* Измерять дома, высота скачет на 3 м от открытия форточки.
* Держать датчик в руке, дыхание и тепло исказят показания.
* Игнорировать QNH, Ошибка до 250 метров.
* Верить последнему знаку, округляй до целых.
